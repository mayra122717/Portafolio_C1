En este tema nos enfocamos en el uso de Componentes y JSX en React para construir interfaces de usuario modulares y dinámicas. Aprendemos a dividir nuestra aplicación en componentes reutilizables (tanto de clase como funcionales) y a utilizar JSX para combinar JavaScript y HTML de manera clara. Exploramos distintas formas de aplicar estilos —hojas CSS, módulos CSS e inline styles— y, además, incorporamos Tailwind CSS en nuestros proyectos React para diseñar rápidamente con clases utilitarias. Vemos cómo transmitir datos entre componentes mediante props y cómo usar children para anidar contenido dentro de ellos. En conjunto, este tema nos permite estructurar la UI de forma escalable, gestionar el flujo de datos y estilizar cada componente de manera flexible, sentando las bases para desarrollar aplicaciones web modernas con React.

¿Qué aprendimos?

  • Renderizar HTML en React: Aprendimos a usar ReactDOM.render() para insertar componentes en el DOM, identificando el punto de montaje en un elemento con id (por ejemplo, root).
  • Componentes en React: Descubrimos qué es un componente, cómo su nombre debe comenzar con letra mayúscula, y su estructura básica (HTML, JavaScript y CSS integrados).
  • Desarrollo de Componentes (Clase y Función): Conocimos las diferencias entre componentes de clase y de función: sintaxis, ciclo de vida básico (para clases) y simplicidad de las funciones.
  • Diseño de Componentes (Estilos): Vimos tres formas de aplicar estilos a componentes React – hojas de estilo CSS, módulos CSS y estilos en línea – y entendimos cuándo y cómo usar cada una.
  • Librería Tailwind CSS en React: Aprendimos a instalar Tailwind CSS en un proyecto React, configurar tailwind.config.js y usar sus utilidades (tailwind base; tailwind components; tailwind utilities;) para diseñar rápidamente.
  • Props y Children: Comprendimos cómo transmitir datos de un componente padre a uno hijo mediante props, y cómo usar children para insertar contenido anidado dentro de componentes reutilizables.

¿Cómo lo aprendimos?

Aprendimos a renderizar componentes usando ReactDOM.render() y verificamos su funcionamiento en el navegador. Dividimos la interfaz en componentes reutilizables, creando archivos .jsx y respetando la convención de nombres en React. Comparando componentes de clase y funcionales, entendimos que los funcionales son más simples y prácticos. Aplicamos estilos usando hojas CSS, módulos CSS y estilos en línea, comprobando sus efectos directamente en los componentes. Además, instalamos y configuramos Tailwind CSS en nuestro proyecto React, editando tailwind.config.js y probando clases utilitarias en el diseño, lo que nos permitió estilizar de forma rápida y flexible.
